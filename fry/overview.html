Overview of Fry

getting started || tutorial
This document describes how to set up, run, and use the interactive interpreter of Fry. See the tutorial for an introduction to the language itself.

Downloading and running the interpreter

Fry comes equipped with an interactive interpreter. It is written in Haskell, so you'll need a Haskell compiler, such as GHC. Once you have GHC, download the sources from Fry's github repository. Go to the "interpreter" directory and compile it with
make
or, if you don't use Makefile,
ghc --make -O2 Main -o fry
As a result, GHC will create a new executable called "fry". Run it, and you will see something like the following:
$ ./fry Welcome to FRY ver. 0.0.0 type :? for help Fry>
Congratulations, you can start your adventure with Fry.

Your first Fry program

Run the interpreter, type in "2 + 2", and press Enter. You should see something like the following:
Fry> 2 + 2 4
Yes, you've just written your first Fry program.

Loading source files

Obviously, you don't have to type your program into the terminal each time. You can save it in a file (the default Fry's extension is .fry) and load it into the interpreter with ":l <filename>". The interpreter will answer with "[]", which means that the operation succeeded. For example, consider the following file, called "fib.fry":
object fibonacci [ x = 0, y = 1, next = this{x = this.y}{y = this.x + this.y}, nth n = if n <= 0 then this.x else this.next.nth (n - 1) ]
You can load and use this file as follows:
Fry> :l fib [] Fry> fibonacci.nth 1 1 Fry> fibonacci.nth 3 2 Fry> fibonacci.nth 7 13
Note that you don't have to type the file extension .fry. The interpreter is smart enough to figure it out.
