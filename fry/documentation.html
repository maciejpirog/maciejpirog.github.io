welcome || core || syntactic sugar
Fry is a programming language that comes equipped with a formal semantics, like, among others, Standard ML (though Fry is a way much simpler language). Generally speaking, it consists of a small core expression language and some syntactic sugar that makes Fry more programmer-friendly. This documentation presents the full syntax of both, the formal semantics of core, and the translation rules from the syntactic sugar constructs into core.
The core language is based on Mart√≠n Abadi and Luca Cardelli's calculus of primitive objects under the call-by-name evaluation strategy. The calculus was extended with features that support object inheritance and side-effects.
The syntactic sugar includes constructs that make Fry more like other object-oriented languages (like chain fields updates, giving names to objects, and inheritance via the extends keyword). Some other powerful constructs can be defined and embedded in Fry, like first-class lambda-expressions.
